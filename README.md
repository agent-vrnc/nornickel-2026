# Научный клубок — Норникель

Поисково-аналитическая система знаний R&D: семантический поиск по корпусу PDF/DOCX-документов, аналитические ответы через YandexGPT, детектор пробелов в исследованиях и граф связей «материалы → процессы → оборудование → результаты».

> ⚠️ Прототип на базе Google Colab. Работает с локальной копией корпуса документов, ключи и пути к данным задаются пользователем, в репозитории не хранятся.

## Возможности

- 📂 Загрузка PDF и DOCX из произвольной вложенной структуры папок
- 🔍 Семантический поиск по корпусу документов (ChromaDB + эмбеддинги)
- 🤖 Аналитические ответы через YandexGPT (можно адаптировать под GigaChat / Mistral)
- 🕸️ Граф связей: материалы → процессы → оборудование → результаты (NetworkX + PyVis)
- 🕳️ Детектор пробелов и противоречий в исследованиях
- 🎛️ Фильтрация по типу документа, источнику, году, географии
- 📤 Экспорт результатов в DOCX
- ⚡ Кэширование распарсенных документов в SQLite (повторный запуск обрабатывает только новые/изменённые файлы)

## Стек

Python · Jupyter/Colab · ChromaDB · pdfplumber / PyMuPDF · python-docx · NetworkX · PyVis · ipywidgets

## Быстрый старт

### Вариант 1: Google Colab (рекомендуется)

1. Откройте `nornickel_rag_system.ipynb` в [Google Colab](https://colab.research.google.com/).
2. Добавьте свой ключ YandexGPT в **Colab Secrets** (значок ключа на левой панели) под именем `YANDEX_API_KEY` и `YANDEX_FOLDER`, либо просто введите их вручную, когда ноутбук запросит — они не сохраняются в файле.
3. Укажите путь к своему архиву документов (переменная `NORNICKEL_DATA_FILE_ID` или ручной ввод в соответствующей ячейке).
4. Выполните ячейки по порядку (`Runtime -> Run all`).

### Вариант 2: локально / Jupyter

```bash
git clone https://github.com/<ваш-логин>/<репозиторий>.git
cd <репозиторий>
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
jupyter notebook nornickel_rag_system.ipynb
```

Ячейка `!pip install ...` и импорт `google.colab` рассчитаны на Colab — при локальном запуске эту ячейку можно пропустить (зависимости уже стоят из `requirements.txt`), а обращения к `google.colab.files` заменить на обычную загрузку файлов с диска.

## Переменные окружения

| Переменная | Назначение | Обязательна |
|---|---|---|
| `YANDEX_API_KEY` | API-ключ YandexGPT | да |
| `YANDEX_FOLDER` | ID каталога в Yandex Cloud | да |
| `NORNICKEL_DATA_FILE_ID` | ID архива с документами на Google Drive | да, если данные хранятся там |

Ключи можно задать через `export YANDEX_API_KEY=...` перед запуском Jupyter, через Colab Secrets, либо ввести вручную — ноутбук запросит их через `getpass`, если переменная окружения не найдена.

## Безопасность и конфиденциальность

- Ноутбук не содержит реальных ключей API — они запрашиваются во время выполнения.
- Не коммитьте в репозиторий папку `data/`, файлы `*.sqlite`, `*.zip` и любые выгрузки корпуса документов — они уже добавлены в `.gitignore`.
- Перед каждым коммитом проверяйте вывод ячеек (`Cell -> All Output -> Clear`) — в выводах может оказаться содержимое обрабатываемых документов.
- Если корпус документов содержит внутреннюю/конфиденциальную информацию компании, держите такой репозиторий **приватным** и не публикуйте ID папок/файлов с реальными данными.
